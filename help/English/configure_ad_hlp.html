Info

Some important points about LDAP/AD configuration

Server URL

Through a URL, the client can communicate with the directory service daemon on the server.

  • - IP Address made of 4 numbers separated by dots as in "xxx.xxx.xxx.xxx".
  • - The prefix can be set as ldap or ldaps to distinguish whether SSL is enabled or not.
  • - 'xxx' ranges from 0 to 255.
  • - First 'xxx' must not be 0.
  • - The port in URL can be left unset so that the default port is used.
  • - Supports Host Name

Bind DN

The Distinguished Name of the server, e.g. "cn=Manager, dc=my-domain, dc=com".

Bind Password

Authentication password for server; the password must be at least 4 characters long.

Searchbase

The searchbase of the server, e.g. "dc=my-domain, dc=com".

User ID Attribute (Optional)

User ID for the AD Server
