KCS 策略控制模式

KCS 策略控制模式允许管理员通过键盘控制器样式 (KCS) 界面控制对 BMC 的访问。 KCS接口是服务器主机(UEFI, 操作系统)用来与服务器BMC进行通信的通信通道。尽管 KCS 接口受到环 0 访问的保护，但它未经身份验证并且对于某些部署模型被认为是不安全的。 KCS 策略控制模式限制由智能平台管理接口 (IPMI) 规范定义的命令在核心 BIOS 功能完成后通过 KCS 接口发送。选项来自：

  • KCS 策略控制模式 - 配置

    • 此配置设置适用于主机操作系统和 BMC 之间的正常 IPMI 兼容通信。配置 BMC 配置以进行部署时，应使用此模式。

  • KCS 策略控制模式 - 已配置主机已禁用

    • 此配置设置禁用主机操作系统和 BMC 之间的 IPMI KCS 命令接口。这是不合规的 IPMI 配置，会影响在主机操作系统上运行的服务器管理软件的操作。这仅适用于 KCS 接口上的 IPMI 命令，不适用于经过身份验证的 BMC 网络接口。

  • KCS 策略控制模式 - 预置主机白名单

    • 此配置设置允许 BMC 固件使用访问控制列表，允许在主机操作系统上执行的应用程序使用 KCS 接口访问一组有限的 IPMI 命令。这是不合规的 IPMI 配置，可能会影响在主机操作系统上运行的服务器管理软件的操作。例如，用于配置 BMC 配置设置或控制主机操作系统状态的 IPMI 命令将在 BMC KCS 接口处于受限模式时被禁止。这仅适用于 KCS 接口上的 IPMI 命令，不适用于经过身份验证的 BMC 网络接口。

主机接口

启用/禁用 eth2 的主机接口。

登录尝试失败

输入失败的登录尝试次数。 设置用户在被锁定之前允许的失败登录尝试次数。零表示没有锁定。失败的登录尝试次数应介于 0 和 255 之间。默认为 3 次尝试。

用户锁定时间（秒）

以秒为单位设置用户在被允许再次登录之前被锁定的时间。零表示禁用用户锁定时间。如果用户由于密码错误阈值而被自动禁用，则该用户将保持禁用状态，直到通过“设置用户访问权限”命令重新启用。用户锁定时间应在 0 到 65535 之间。默认为 60 秒。

密码规则设置 - 复杂性

选择密码复杂度下拉菜单以设置密码复杂度（低、中、高）。

低
-密码可以由字符a-zA-Z0-9_组成，包括标点符号
-密码长度不能少于8个字符
-pam_cracklib 用于在接受密码之前对密码进行强度检查。 密码不能有字典词、回文等。
中等
-仅包含可打印字符
-密码长度不能少于8个字符
-不能包含用户名或用户名倒序（不区分大小写）。
-满足以下 4 个条件：
1. 包含大写
2. 包含小写
3. 包含数字数字
4. 至少包含一个空格或符号
高
-仅包含可打印字符
-密码长度不能少于8个字符
-满足以下 4 个条件：
1. 包含大写
2. 包含小写
3. 包含数字数字
4. 包含符号
-不能包含用户名或用户名倒序（不区分大小写）。
-不能包含超过 3 个字符的连续数字（例如 BMC(123)ste、BMC(654)sfc 是不允许的）
-不能包含长度超过 3 个字符的连续字母（例如 (AbC)3478!、57$(DeF)68k 不是允许））
-字母不能超过 3 个字符（例如 (Fher)145! 不允许））
-数字不能超过 3 个字符（例如 Fgke(1245)#@, (4390)FGL$ 是不允许的）
-包含不超过 3 个相同的字母（不区分大小写）（例如 Fkr4fcpF&f (4 'f' 或 'F')、Glg5gt2G! (4 'g' 或 'G') 是不允许的）
-对于符号，请改用 ASCII 代码。 连续 ASCII 码中的字母/数字/符号组合不能长于或等于 3 个字符（例如，以下不允许）
  #$%, ASCII 35, 36, 37 <- 符号
  9:;, ASCII 57, 58, 59 <- 数字 + 符号
  YZ[, ASCII 89, 90, 91 <- 字母 + 符号
-符号不能超过 3 个字符（例如，#$&; 是不允许的）

密码规则设置 - 密码历史

选择密码历史下拉菜单设置密码历史(0到5)。
